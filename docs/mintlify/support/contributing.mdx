## Current State

Mori is in the early stages of development. For now, the best way to contribute is by **opening issues** — bug reports, feature requests, edge cases you encounter, or suggestions for improvement.

<Card title="Open an Issue" icon="github" href="https://github.com/psrth/mori/issues/new">
  github.com/psrth/mori/issues
</Card>

## Why Issues First

As the repo stabilizes, we'll have more clarity on how to moderate open-source contributions in a world where code is heavily AI-assisted. Until then, issues give us the signal we need to prioritize the right work.

Your feedback directly shapes what gets built next. Every bug report, every edge case, every "this didn't work the way I expected" — it all matters.

## What Helps Most

- **Bug reports** with reproduction steps and proxy logs
- **Edge cases** from real-world usage (especially with ORMs, migration tools, or AI agents)
- **Feature requests** that describe the use case, not just the feature
- **Engine/provider gaps** — databases or platforms you need that aren't supported yet
