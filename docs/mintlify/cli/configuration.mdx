## The `.mori/` Directory

Mori stores all state in a `.mori/` directory at your project root. This directory is created by `mori init`.

```
.mori/
  mori.yaml             # Connection config (engine, addresses, ports)
  shadow/               # Shadow container metadata
  state/
    delta.json           # Delta Map (locally modified rows)
    tombstones.json      # Tombstone Set (locally deleted rows)
    schema_registry.json # Schema divergence tracking
    tables.json          # Table metadata (PKs, types)
    sequences.json       # Sequence offsets
  log/                   # Query and routing logs
```

## File Reference

### `mori.yaml`

Connection configuration — engine type, Prod address, Shadow address, port, credentials, provider settings.

<Tip>
You may want to check `mori.yaml` into version control (with secrets excluded) so teammates can share the same connection setup.
</Tip>

### `shadow/`

Metadata for the Shadow database container (Docker container ID, port mappings, image version). Managed automatically by Mori.

### `state/delta.json`

The Delta Map — a set of `(table, primary_key)` pairs identifying locally modified rows. When a row is in the Delta Map, Shadow holds the authoritative version.

### `state/tombstones.json`

The Tombstone Set — a set of `(table, primary_key)` pairs identifying locally deleted rows. Tombstoned rows are filtered from Prod results during merged reads.

### `state/schema_registry.json`

Schema divergence tracking. Records all DDL changes made on Shadow: added columns, dropped columns, renames, type changes, new tables, dropped tables.

### `state/tables.json`

Table metadata cached from Prod: primary key columns, primary key types, column lists. Used by the Classifier and Router to make routing decisions.

### `state/sequences.json`

Sequence offsets per table. Ensures Shadow-generated primary keys don't collide with Prod's existing or future PKs.

### `log/`

Query and routing logs. View with `mori log` or the TUI dashboard (`mori dash`).
