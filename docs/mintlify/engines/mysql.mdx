## Overview

| Property | Value |
|----------|-------|
| Engine ID | `mysql` |
| Wire Protocol | MySQL wire protocol |
| Default Port | 3306 |
| Status | Supported |
| Classifier | AST-based (`vitess/sqlparser`) |
| Shadow | Docker container (version-matched) |

MySQL uses its native wire protocol. Mori handles the full MySQL handshake, authentication, and command protocol including `COM_QUERY` and `COM_STMT_*` prepared statement messages.

## Connection Parameters

| Field | Description | Default |
|-------|-------------|---------|
| `host` | Database host | -- |
| `port` | Database port | `3306` |

| `database` | Database name | -- |
| `ssl_mode` | SSL mode | -- |

## Implementation Details

### Query Classification

Uses `vitess/sqlparser` (from the Vitess project) for full AST-based classification. This parser understands MySQL-specific syntax including:

- Standard DML (SELECT, INSERT, UPDATE, DELETE)
- MySQL-specific constructs (INSERT ... ON DUPLICATE KEY UPDATE, REPLACE INTO)
- DDL (CREATE, ALTER, DROP, RENAME)
- Transaction control (BEGIN, COMMIT, ROLLBACK, START TRANSACTION)
- SHOW commands and MySQL system queries

### Shadow Strategy

Docker container running MySQL, version-matched to production. The version is auto-detected via `SELECT VERSION()` during `mori init`.

- Image: `mysql:<version>` (matched to prod)
- Schema is dumped using `mysqldump` and replayed on shadow
- Auto-increment offsets are adjusted to avoid PK collisions

### Extended Query Protocol

Supports MySQL prepared statements (`COM_STMT_PREPARE`, `COM_STMT_EXECUTE`, `COM_STMT_CLOSE`) with parameter binding and cursor-based result fetching.

## Example Usage

```bash
# Initialize with MySQL
mori init --from "mysql://user:pass@prod-host:3306/mydb"

# Start the proxy
mori start

# Connect with the MySQL client
mysql -h 127.0.0.1 -P 3306 -u user -p mydb
```

## Known Limitations

<Warning>
**Merged read breaks aggregate-only queries with `only_full_group_by`** -- When Mori performs a merged read on `SELECT AVG(col) FROM table`, it injects non-aggregated columns (for delta patching), which violates MySQL's `only_full_group_by` SQL mode. This affects aggregate-only queries without a GROUP BY clause.

Workaround: MariaDB is unaffected because it does not enforce `only_full_group_by` by default. For MySQL, you can disable the mode with `SET sql_mode = ''` in your session.
</Warning>
