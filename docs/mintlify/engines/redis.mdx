## Overview

| Property | Value |
|----------|-------|
| Engine ID | `redis` |
| Wire Protocol | RESP (Redis Serialization Protocol) |
| Default Port | 6379 |
| Status | Supported |
| Classifier | Command-based (lookup table) |
| Shadow | Docker container (version-matched) |

Redis uses the RESP protocol. Mori implements a RESP proxy that intercepts and classifies every Redis command, routing reads to prod and writes to shadow.

## Connection Parameters

| Field | Description | Default |
|-------|-------------|---------|
| `host` | Redis host | -- |
| `port` | Redis port | `6379` |
| `password` | Password | -- |
| `db_number` | Database number | `0` |
| `ssl` | Enable SSL/TLS | `false` |

## Implementation Details

### Query Classification

Uses a command-based classifier with a static lookup table mapping every Redis command to its operation type. This covers 100+ commands across all Redis data types:

- **Strings**: GET, SET, MGET, MSET, INCR, APPEND, etc.
- **Hashes**: HGET, HSET, HGETALL, HMGET, HDEL, etc.
- **Lists**: LPUSH, RPUSH, LPOP, LRANGE, LLEN, etc.
- **Sets**: SADD, SREM, SMEMBERS, SINTER, SUNION, etc.
- **Sorted Sets**: ZADD, ZRANGE, ZSCORE, ZRANK, etc.
- **Streams**: XADD, XREAD, XRANGE, XLEN, etc.
- **Key management**: DEL, UNLINK, RENAME, EXPIRE, TTL, etc.
- **Transactions**: MULTI, EXEC, DISCARD, WATCH
- **Server**: PING, INFO, CONFIG, AUTH, SELECT

Each command is classified as read, write, DDL (FLUSHDB, FLUSHALL), or transaction control.

### Shadow Strategy

Docker container running Redis, version-matched to production. The version is auto-detected from the `INFO` command during `mori init`.

- Image: `redis:<major>.<minor>` (matched to prod)
- No schema dump needed -- Redis is schemaless
- Keys sampled during init for metadata

### Delta Tracking

Redis delta tracking works at the key level. When a write command targets a key, that key is added to the delta map. Subsequent reads for delta keys are routed to shadow.

For multi-key commands (MGET, DEL, etc.), each key is tracked independently.

### Supported Data Types

All Redis data types are supported: strings, hashes, lists, sets, sorted sets, and streams. The proxy correctly routes commands for each type.

### AUTH Support

When the production Redis requires authentication, Mori handles AUTH transparently. The proxy authenticates to both prod and shadow connections independently.

## Example Usage

```bash
# Initialize with Redis
mori init --from "redis://:password@prod-redis:6379/0"

# Start the proxy
mori start

# Connect with redis-cli
redis-cli -h 127.0.0.1 -p 6379
```

## Known Limitations

**RENAME delta tracking misses destination key** -- After `RENAME src dst`, the proxy tracks the delta for `src` (deleted) but doesn't register `dst` as a new shadow key. A subsequent `GET dst` reads from prod instead of returning the renamed value. This is a very low-impact edge case.
