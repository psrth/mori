## Overview

Mori includes an MCP (Model Context Protocol) server that enables AI coding agents to interact with databases through Mori's safe proxy layer.

The MCP server exposes a `db_query` tool over HTTP. AI agents connect to this endpoint, issue database queries, and receive results as JSON — all routed through Mori's copy-on-write proxy.

## How It Works

1. The MCP server starts alongside the Mori proxy
2. AI agents connect to the MCP HTTP endpoint
3. The server forwards queries to Mori as a standard database client
4. Mori routes reads to Prod, captures writes in Shadow
5. Results are returned as JSON to the agent

## Use Cases

<CardGroup cols={2}>
  <Card title="Safe Production Queries">
    AI agents can query real production data without any risk of modification. All reads go through Mori's read-only Prod connection.
  </Card>
  <Card title="Risk-Free Mutations">

  </Card>

## Configuration

The MCP server port is configurable. The endpoint is available at `/mcp` on the configured port.

```bash
# Start Mori with MCP server
mori start
```

<Info>
The MCP server connects to Mori's proxy as a standard PostgreSQL client. From the MCP server's perspective, Mori is just a database — it doesn't need to know about the Prod/Shadow split.
</Info>
