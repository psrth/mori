## memento mori

*Remember that you must die.*

The shadow database is ephemeral. Every write, every schema change, every local mutation — it all exists only until you reset. The proxy must die. That impermanence is the point: you can break anything, explore freely, and start over in seconds.

## irom

Flip "mori" and you get **irom** — *it ran on my machine*.

The classic developer excuse. Your code works locally against seed data, but fails in production because the real data looks different. Mori solves this by giving you the real data — safely, locally, without risk.
