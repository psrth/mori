## Overview

| Property | Value |
|----------|-------|
| Provider ID | `direct` |
| Compatible Engines | All 9 engines |
| SSL Default | Engine default |
| Extra Fields | None |

The Direct provider is the fallback for any database not hosted on a specific managed platform. Use this when connecting to self-hosted databases, on-prem servers, or any provider not listed in Mori's provider list.

## How It Works

The Direct provider passes your connection parameters through without modification. No SSL override is applied -- it uses whatever SSL mode you configure (or the engine's default).

## When to Use

- Self-hosted PostgreSQL, MySQL, Redis, etc.
- Docker or Kubernetes-hosted databases
- Local development databases
- Any provider not explicitly supported by Mori

## Example

```bash
# Interactive setup
mori init
# Select "Direct / Self-Hosted" when prompted for provider

# Connection string
mori init --from "postgres://user:pass@my-server:5432/mydb"
```
